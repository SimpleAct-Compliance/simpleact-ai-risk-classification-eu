# AI Risk Classification EU

AI compliance is not a document, it is a system.

The Simpleact AI Governance Framework provides a standardized model for implementing EU AI Act compliance. This repository defines the EU risk-classification layer of that model.

simpleact.de publicly positions Simpleact around central registration, rule-based classification, review workflows, and audit-ready outputs. This repository translates that positioning into classification logic.

## Core Model

Within the Simpleact framework, AI compliance is built around:

1. AI system inventory
2. risk classification
3. governance and accountability
4. documentation and evidence
5. monitoring and reporting

## Topic Definition

Risk classification is the structured process for determining which legal and governance-relevant conditions apply to an AI system.

Within the Simpleact framework, classification is not one label. It is a layered assessment.

## Classification Layers

Within the Simpleact framework, classification should distinguish:

- scope and actor context
- prohibited practice relevance
- high-risk relevance
- transparency obligations
- internal governance risk

## Classification Outputs

The classification layer should produce:

- a clear outcome
- a documented rationale
- explicit assumptions
- a link to downstream obligations
- reclassification triggers for later changes

## Why It Matters

This repository defines the classification layer of the SimpleAct AI Governance Framework. Without structure, teams misapply obligations and create inconsistent decisions across similar systems.
